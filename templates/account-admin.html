重要提示 + 「零信任客户端」声明： 本工具只生成哈希（单向不可逆）。SHA-256 设计上就不能从哈希反推回明文密码。
发给用户的应该是明文用户名 + 密码，不是哈希。
客户端校验可被绕过：本方案依赖 localStorage.unlocked=true，任何持有浏览器 dev tools 的人都能改 —— 本工具用于软门 / 付费墙，不替代真鉴权（真鉴权请加 Node Functions 后端，见 SKILL.md 第五节）。

使用流程：
1. 客户付款 元 → 加 微信 收款
2. 打开本页面 → 「批量生成」标签 → 输入 用户名:密码（一行一个）
3. 点击「生成完整 accounts.json」→ 复制 JSON → 覆盖到 /accounts.json
4. 把「用户名 + 密码」发给客户（明文，不是哈希）
5. 客户登录失败？→「验证」标签 → 输入客户反馈的密码 + accounts.json 里的哈希 → 确认是否一致

工具特性：纯前端 SHA-256 · 离线可用 · 不上传任何数据 · 盐值必须与主 app 一致
