Important + "zero-trust client" notice: This tool only generates hashes (one-way, irreversible). SHA-256 cannot recover the plaintext password from a hash by design.
Send your users the plaintext username + password, not the hash.
Client-side checks can be bypassed: this pattern depends on localStorage.unlocked=true, which anyone with browser dev tools can flip —— This tool implements a soft gate / paywall, not real auth (for real auth, add a Node Functions backend — see SKILL.md §5).
